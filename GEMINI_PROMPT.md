## 正在设计：外部情报系统

### 已完成的设计决策（grill-me 确立）

**数据源**：akshare 宏观/新闻接口 + NewsAPI（全球视角）+ 财联社 RSS（专业财经深度）

**过滤架构**：两阶段
- 阶段 1：结构化规则粗筛（去重 + 去纯噪音如娱乐/体育）
- 阶段 2：DeepSeek 精读摘要 + 推理 + 置信度评分

**三通道进入阶段 2**：
- 🅰️ 宏观必过：利率/准备金/CPI/PMI/重大政策 → 全部放行
- 🅱️ 画像命中：标题命中持仓/行业/关注词 → 标记高相关，优先排序
- 🅲️ LLM 判断：边界模糊但可能有间接影响 → DeepSeek 自行判断

**画像定位**：不是门禁卡，是信号增强器。不砍宏观信息，只做高亮和排序。

**双通道输出**：企微推送（每日 3 条简报）+ 家庭看板展示（全部简报可浏览）

**推送策略**：高置信度主动推企微，中低置信度放看板

## 请求建议

### 问题 1：DeepSeek 阶段 2 的输出结构

阶段 2 每天处理 ~15-30 条过滤后的新闻。DeepSeek 需要输出结构化结果。

目前草稿：

```json
{
  "briefs": [
    {
      "title": "央行降准 0.5 个百分点",
      "summary": "...",
      "relevance": "高",
      "reasoning": "影响你持有的银行股和房贷利率",
      "impacted_members": ["dad", "me"],
      "suggested_action": "关注下周 LPR 报价",
      "actionability": "监控"
    }
  ],
  "market_sentiment": "偏多",
  "top_3": [0, 3, 7]
}
```

问题：
1. 这个输出结构够不够？缺什么字段？
2. actionability 的三个等级（"立即行动" / "监控" / "无需行动"）是否合理？
3. 推送阈值怎么定？「高置信度主动推送」是指 relevance=高 还是 actionability=立即行动？
4. 每日本简报是否需要给每条打置信度分数（1-10）还是用「高/中/低」就够？
5. 有什么我没意识到的设计问题？

### 问题 2：数据获取的具体接口

akshare 的新闻类接口很多，以下哪些最有价值：
- `ak.stock_info_global_ths()` — 同花顺全球财经
- `ak.stock_zh_bond_spot_em()` — 利率/债券
- `ak.macro_china` 系列 — CPI/PMI/货币供应
- `ak.news_economic_baidu()` — 百度财经新闻
