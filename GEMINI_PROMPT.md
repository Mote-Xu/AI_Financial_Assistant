## 下一步：家庭版财务分析

### 需求

**以家庭为整体，每个成员保持独立财务数据，同时生成家庭汇总视图。**

具体：
1. 我、爸爸、妈妈各自有独立的 `assets.md`、`income.md`、`insurance.md`
2. 家庭汇总：合并总资产、合并现金流、家庭保障缺口（谁没投保？）
3. 分析报告：月体检从"我个人"升级为"全家资产配置+现金流+风险"
4. 爸妈微信只看到汇总和自己那部分

### 初步设想的数据结构

```
family/
  members.json          ← 定义成员 {me, dad, mom}
  me/
    assets.md           ← 我的数据（已有）
    income.md
    insurance.md
  dad/
    assets.md           ← 爸爸的数据
    income.md
    insurance.md
  mom/
    assets.md           ← 妈妈的数据
    income.md
    insurance.md
  _family/
    snapshot.md         ← 自动汇总
    analysis_*.md       ← 家庭分析报告
```

### 附加问题：成员隐私隔离

**我的情况**：学生身份，无稳定收入。父母收入和我的财务状况性质不同。我希望：

- 我看得到自己的资产，也看得到**家庭汇总**（总资产/总负债/总现金流），但**不应该直接看到**父母的详细收入条目和具体金额构成
- 爸妈各自看到自己的详细数据 + 家庭汇总
- 数据库层面数据在一起（SQLite），但**视图层做权限过滤**

**初步隐私模型**：
```
family/                         ← 统一数据目录
  members.json                  ← {me: {role:student}, dad:{}, mom:{}}
  summary/                      ← 自动汇总（所有人可看）
  me/assets.md income.md ...    ← 仅我可看详细
  dad/assets.md income.md ...   ← 仅爸爸可看详细
  mom/assets.md income.md ...   ← 仅妈妈可看详细
  _db/family.db                 ← SQLite，所有数据
```

**请求**：作为学生+家庭成员这种特殊角色，数据隐私的边界怎么划最合理？有什么行业内家庭理财系统的多用户隐私实践参考？
